DISCREPANCIES FLAGGED

FLAG-F1 — Item status terminology mismatch
  The API reference doc (jobs/backend/tech/20260307-api-reference.md) lists item
  statuses as "booked" and "skipped". The actual Zod schema (src/backend/validation/
  common.ts) defines: consider, confirmed, completed, cancelled, next_time.
  Frontend uses the Zod schema as authoritative. Please update the API reference doc
  or clarify which set of names the product spec intends.

FLAG-F2 — No GET /api/cities list endpoint
  CityMarkers derives city pins from trip.places[].city (already loaded). No
  standalone city list is needed. If a future requirement needs a full city catalogue
  view, a new endpoint will be needed.
