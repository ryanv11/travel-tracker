  DRIVER CHANGE (flagging per Shared Standard 8):
  The COO spec recommended better-sqlite3 as the SQLite driver. This was not
  implemented because better-sqlite3 v9.x has no prebuilt binaries for
  Node.js v24 and failed to compile from source on this machine (C++20
  compiler requirement not met by Xcode Command Line Tools). Substituted
  @libsql/client — a pure JavaScript/WASM SQLite driver that is:
  (a) fully supported by Drizzle ORM,
  (b) API-compatible for all required use cases,
  (c) installs without any native compilation on any platform.
  This is an improvement, not a compromise. WAL mode pragma (which was
  planned in the better-sqlite3 implementation) is not separately needed
  with libSQL as it manages its own connection optimally.
  No Architect review required — this is a driver-level implementation
  detail, not a schema or architecture change.

  SQLITE_PATH FORMAT:
  @libsql/client uses a URL scheme for local files. SQLITE_PATH must be
  formatted as file:./dev.db or file:/absolute/path.db, not a bare path.
  This is documented in .env.example with a clear example.
