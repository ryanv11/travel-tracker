---

DESIGN DECISIONS THE PO SHOULD BE AWARE OF

1. Audit findings addressed in both mockups:
   - T-02 (filter bar too tall): Filters are now a compact single row (Option A) or
     chips strip (Option B), not two stacked rows
   - T-03 (cards too sparse): Card padding is 12px with 4–8px gap, not 16px/12px
   - T-04 (Edit button equal weight to status badge): Edit is a small outline button,
     badge is the primary status signal
   - TD-01/T-05 (raw ISO dates): All dates formatted as "15 Mar 2026 – 24 Mar 2026"
   - G-04 (emoji icons): No emoji used for icons; type identifiers use single letters
     (H = Hotel, R = Restaurant, T = Experience) as placeholder stand-ins for Lucide icons
   - M-02 (no map legend): Option B map state shows a simulated legend overlay
   - RP-04 (red lock button): Status transition uses green (emerald) for the positive
     "Move to Review" action, not red

2. The lock icon (&#128274;) is used on the "Locked" badge as it is a recognised
   semantic symbol, not decorative. It will be replaced by Lucide Lock icon in
   the actual implementation.

3. Country full names are used in place of country_code in PlaceSection subtitles
   (e.g. "Ireland" not "IE") — this addresses G-09.

4. Option B uses a sticky 48px nav bar and overflow:hidden on the app shell to keep the
   two panels at full viewport height with independent scroll — this is the correct
   pattern for a data-dense desktop application.

5. Status transitions in Option B are shown as a slim strip between the header and the
   itinerary content. This keeps them accessible but not visually dominant.

6. The "Add Item" button inside each PlaceSection is an outline button (not filled
   primary), addressing TD-07. The only filled primary button is "New Trip" / "+ New"
   at the list level.
