SUMMARY OF DECISIONS

ROOT CAUSE:
  The regions table has no iso_3166_2 column. The GeoJSON source uses
  promoteId="iso_3166_2" so feature-state IDs must be ISO 3166-2 strings
  (e.g. 'US-CA'). RegionLayer.tsx was using integer region_id — never matches.

SOLUTION (Option A — as directed):
  1. Add iso_3166_2 TEXT NOT NULL to regions table with UNIQUE index
  2. Populate all 72 seeded regions with correct ISO 3166-2 codes
  3. Backend returns iso_3166_2 in both region shading endpoints
  4. Frontend uses region.iso_3166_2 as the setFeatureState id

iso_3166_2 is NOT NULL. A nullable column would allow the silent mismatch
to recur for any region missing a code. Placeholder 'XX-UNKNOWN' required
for any user-created region without a known code.
