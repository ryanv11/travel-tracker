MAJOR BUGS (4) — BLOCK QA SIGN-OFF

BUG-01: MAJOR — City markers never appear on map (AC-05 complete failure)
  MapPage passes TripSummary[] (no places data) to CityMarkers. The component
  uses a type-cast hack that always falls back to empty array. No city pins
  will ever render. Requires a cross-team solution — options:
  (a) New GET /api/cities/resolved endpoint (BACKEND), or
  (b) Modified MapPage to load full trip details, or
  (c) Include minimal city data in the TripSummary response.
  Needs COO direction on which approach BACKEND should implement.

BUG-02: MAJOR — Region shading completely broken (MP-02 complete failure)
  RegionLayer sets feature-state using integer DB region_id values, but
  the GeoJSON (regions.json) uses ISO 3166-2 string codes (e.g. "US-CA")
  as feature IDs. These never match. No region will ever be shaded.
  Root cause: the regions table has no iso_3166_2 column. This is a schema
  defect — requires DATABASE migration + BACKEND API change + FRONTEND update.
  Needs Architect review and COO sign-off before work begins.

BUG-03: MAJOR — Carry-forward modal never shown (AC-12, IT-07 complete failure)
  Race condition in AddPlaceFlow: useEffect fires before the carry-forward
  candidates API query resolves, calling onClose() with an empty candidates
  list. Modal is never displayed. IT-07 is completely broken.
  Fix: check isFetched/isLoading before acting on empty candidates list.
  FRONTEND-only fix, small change.

BUG-04: MAJOR — No UI path to revert trip from review_pending to planning (AC-08, AC-13)
  TripDetailPage renders ReviewPanel for review_pending trips. ReviewPanel
  has no "Return to Planning" button. The valid backend transition
  review_pending → planning is inaccessible from the UI.
  Fix: add "Return to Planning" button + confirm dialog to ReviewPanel.
  FRONTEND-only fix, small change.
