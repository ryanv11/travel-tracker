## Task Checklist

### Task 1 — Auth Middleware (`src/backend/middleware/auth.ts`)
**YES — implemented.**

- `jose` installed (`npm install jose` — was not previously in package.json)
- `requireAuth` middleware: extracts Bearer token, verifies via `createRemoteJWKSet` + `jwtVerify`, resolves internal user, attaches `req.user = { id, clerkId, email }`, calls `next()`
- Returns `401 Unauthorized` on missing token, invalid JWT, expired token, or DB error
- JWKS lazy-initialized on first request (avoids module-load throw in test environment)
- Hard rule respected: zero `@clerk/*` imports anywhere in backend code
- `authenticate` alias exported for backward compatibility (deprecated)

### Task 2 — Users Repository (`src/backend/repositories/users.ts`)
**YES — implemented.**

- `findOrCreateByClerkId(clerkId, email): Promise<User>`
- SELECT by clerk_id → if found, return (update email if changed)
- If not found: INSERT with `crypto.randomUUID()` for UUID v4, return new row
- No extra package needed — Node built-in

### Task 3 — Protect All API Routes
**YES — all `/api/*` routes protected.**

`app.use('/api/', requireAuth)` in both `server.ts` and `server-test-app.ts`.
`GET /health` remains public (registered separately, no auth middleware applied).

### Task 4 — Wire User ID Into Route Handlers
**BLOCKED — cross-team dependency.**

The `trips` table has no `user_id` column. The Database agent's migration (0002) added the `users` table but did not add `user_id` FK to `trips`. Without this column, `db.select().from(trips).where(eq(trips.userId, userId))` cannot compile.

This filter cannot be implemented without a schema migration from the Database agent. The auth middleware correctly attaches `req.user` — the filter can be added once the column exists.
