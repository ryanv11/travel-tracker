KNOWN ISSUES / TECH DEBT
================================================================================

1. NPM AUDIT: 4 moderate vulns in drizzle-kit (dev dep only — esbuild dev server
   CORS issue GHSA-67mh-4wv8-2f99). Zero HIGH/CRITICAL. Defer upgrade until
   drizzle-kit changelog confirms compatibility.

2. CARRY-FORWARD EXECUTION: No POST endpoint to execute carry-forward. Service
   function exists (executeCarryForward in items.service.ts). Awaiting COO
   direction. Documented in completion report.

3. CITY PATCH: No PATCH /api/cities/:id endpoint. Mentioned in corrections
   ("region_id can be set later via PATCH"). Awaiting COO direction.

4. PLACES ACTIVITIES QUERY BUG (cosmetic): The GET /api/trips/:tripId/places
   handler has a double-query pattern for activities that runs two DB queries
   when placeIds.length > 1. The second query uses eq(tripPlaces.tripId, tripId)
   but tripPlaces is not in the FROM clause of that query — this should use
   inArray(tripPlaceActivitiesMap.tripPlaceId, placeIds). Works in practice
   because the second query runs only for >1 places. Could be refactored to a
   single clean inArray query. Not blocking.
