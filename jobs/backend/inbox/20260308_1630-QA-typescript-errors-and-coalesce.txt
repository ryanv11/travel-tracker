BUG-07: Backend TypeScript — 207 errors, no type-check script

Running tsc against the backend source reveals 207 type errors that are
invisible during normal development. `npm run type:check` only covers the
frontend. The main tsconfig.json fails before reaching backend code due to
a config defect (see BUG-09 below).

Error breakdown:
  108 x TS2349 — Drizzle union type: db.select/insert/update not callable
   39 x TS2554 — Drizzle argument count mismatch (same root cause)
   27 x TS2345 — Express v5: req.params is string|string[], not string
   22 x TS7006 — implicit any in .map() callbacks
    1 x TS2305 — coalesce imported from drizzle-orm but doesn't exist

The Drizzle errors (TS2349/2554) stem from AppDatabase being typed as
LibSQLDb | PgDb union. TypeScript cannot resolve overloaded Drizzle methods
across the union. Runtime is unaffected (tsx doesn't enforce types), but
type safety for all db calls is zero.

Action required:
1. Add tsconfig.backend.json and type:check:backend npm script
2. Fix the Drizzle union type — consider typing AppDatabase as any with a
   comment, or use overload narrowing helpers
3. Add `as string` casts on req.params.* accesses (Express v5 pattern)
