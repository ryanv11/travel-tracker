================================================================================
DECISIONS MADE THIS THREAD
================================================================================

  DECISION: Use @libsql/client instead of better-sqlite3
  REASON: better-sqlite3 v9.x has no prebuilt binaries for Node.js v24
  and fails to compile from source (C++20 requirement not met by Xcode
  Command Line Tools on this machine).
  ALTERNATIVE: @libsql/client — pure JavaScript/WASM, no compilation,
  fully supported by Drizzle ORM, identical query API.
  IMPACT: SQLITE_PATH must use libSQL URL format (file:./path.db).
  This is documented in .env.example and in index.ts docstring.
  FLAGGED TO COO: Yes — noted in completion report.
  ARCHITECT REVIEW NEEDED: No — this is a driver implementation detail,
  not a schema or architecture change.

  DECISION: Use tsx instead of ts-node for seed script execution
  REASON: ts-node ESM loader is broken on Node.js v24. tsx is the
  recommended modern replacement and handles TypeScript ESM correctly.
  IMPACT: package.json db:seed script uses tsx.
