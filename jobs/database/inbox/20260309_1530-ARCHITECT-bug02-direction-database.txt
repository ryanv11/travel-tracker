PART 1 — REGIONS TABLE: iso_3166_2 COLUMN (BUG-02)

WHAT:
  Add column  iso_3166_2  TEXT  NOT NULL  to the regions table.

CONSTRAINT:
  NOT NULL. Every seeded region must receive a correct ISO 3166-2 code.
  A nullable column would allow silent mismatches — this is the exact failure
  mode we are fixing.

  If a user-created region (i.e. a region added via the app, not seeded) has
  no ISO code, use the placeholder 'XX-UNKNOWN'. This keeps NOT NULL
  enforceable while making unknowns visible.

INDEX:
  Add a UNIQUE index on iso_3166_2. Each code must appear at most once.
  Name it: uniq_regions_iso_3166_2

SCHEMA CHANGE (add to regions table in schema.ts):
  iso3166_2: text('iso_3166_2').notNull(),

  And in the table options:
  uniqueIndex('uniq_regions_iso_3166_2').on(t.iso3166_2)

MIGRATION SQL (raw, for drizzle-kit migration file):
  ALTER TABLE regions ADD COLUMN iso_3166_2 TEXT NOT NULL DEFAULT 'XX-UNKNOWN';
  CREATE UNIQUE INDEX uniq_regions_iso_3166_2 ON regions (iso_3166_2);

  Then UPDATE statements to populate correct codes (see seed data below).
  After UPDATE, the DEFAULT clause on the column can remain — it covers any
  future user-created regions until a proper geocoding integration is added.
