## TASK 1 — CREATE USERS TABLE

Add to `src/backend/db/schema.ts`:

```ts
export const users = sqliteTable('users', {
  id: text('id').primaryKey(),           // UUID v4 — generated by backend on first sign-in
  clerkId: text('clerk_id').notNull().unique(),  // Clerk user ID (e.g. user_2abc...)
  email: text('email').notNull(),
  createdAt: integer('created_at', { mode: 'timestamp' }).notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp' }).notNull(),
});
```

Notes:
- `id` is a UUID v4 string, NOT an integer autoincrement — per ADL-16 (prevents enumeration)
- `clerk_id` is the external Clerk identifier; the internal `id` is what all FK relationships use
- No `password_hash`, no `oauth_provider`, no `refresh_token` — Clerk handles all of that
- `email` is stored for display/admin purposes; Clerk is authoritative for identity
