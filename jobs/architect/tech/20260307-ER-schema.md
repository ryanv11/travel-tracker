# Travel Tracker — Entity-Relationship Schema
**Version:** 1.1
**Date:** 2026-03-07
**Author:** Architect
**Status:** Approved for DATABASE implementation

---

## Design Decisions (Summary)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Item type-specific fields | Option B: base table + per-type extension tables | Clean type enforcement, indexable rating columns, identical behaviour in SQLite and PostgreSQL. Option C (JSON) rejected because rating sort/filter (IT-08, IT-09) requires column-level indexes, which JSON path queries cannot provide efficiently in SQLite. |
| Place entity | trip_places junction with own PK | Items and activities reference trip_place_id — a stable ID is required. City-level identity is on cities.id, enabling cross-trip queries (IT-07, IT-09). |
| Shading state storage | Computed — not stored | See shading spec. Any stored state would be a denormalisation risk — it would go stale the moment a trip status changed. |
| Soft-delete pattern | is_active flag on all admin list tables | Deactivated items preserved on existing records (AD-06); excluded from entry form queries via WHERE is_active = 1. |
| Timestamp type | TEXT in SQLite (ISO 8601), TIMESTAMP in PostgreSQL | ORM handles the abstraction. All DATETIME columns in schema map to the correct native type per engine. |

## 1. Geographic Hierarchy

### 1.1 countries

Seeded at first launch from bundled data (GE-04, GE-05, GE-06). User can override `region_tier_enabled` and `region_tier_label` per country via admin panel (GE-07, AD-05).

```sql
CREATE TABLE countries (
    country_code          TEXT PRIMARY KEY,     -- ISO 3166-1 alpha-2 e.g. 'US', 'AU', 'FR'
    name                  TEXT NOT NULL,
    region_tier_enabled   INTEGER NOT NULL DEFAULT 0  CHECK (region_tier_enabled IN (0, 1)),
    region_tier_label     TEXT,                -- 'State' | 'Province' | 'Territory' | NULL
    created_at            TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at            TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);
```

**Notes:**
- `region_tier_label` is NULL when `region_tier_enabled = 0`.
- This table doubles as the country configuration store (GE-07). No separate config table needed.
- Boundary polygon GeoJSON for map shading is served as static files (see tech blueprint), not stored in this table.

---

### 1.2 regions

Only populated for countries where `region_tier_enabled = 1`. (GE-01, GE-02, GE-03)

```sql
CREATE TABLE regions (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    country_code  TEXT NOT NULL REFERENCES countries(country_code),
    name          TEXT NOT NULL,               -- e.g. 'California', 'New South Wales'
    created_at    TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at    TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);

CREATE INDEX idx_regions_country ON regions(country_code);
```

---

### 1.3 cities

Leaf level of geographic hierarchy. Coordinates populated via Nominatim geocoding (GE-11, GE-12, GE-13).

```sql
CREATE TABLE cities (
    id                    INTEGER PRIMARY KEY AUTOINCREMENT,
    country_code          TEXT NOT NULL REFERENCES countries(country_code),
    region_id             INTEGER REFERENCES regions(id),   -- NULL when region tier disabled
    name                  TEXT NOT NULL,
    latitude              REAL,                             -- NULL when geocode_status = 'pending'
    longitude             REAL,                             -- NULL when geocode_status = 'pending'
    geocode_status        TEXT NOT NULL DEFAULT 'pending'
                              CHECK (geocode_status IN ('pending', 'resolved')),
    geocode_attempted_at  TEXT,                             -- ISO 8601; timestamp of last attempt
    created_at            TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at            TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);

CREATE INDEX idx_cities_country   ON cities(country_code);
CREATE INDEX idx_cities_region    ON cities(region_id);
CREATE INDEX idx_cities_geocode   ON cities(geocode_status)
    WHERE geocode_status = 'pending';  -- partial index for the geocoding queue
```

**Notes:**
- `region_id` is NULL for cities in countries where `region_tier_enabled = 0`.
- The partial index on `geocode_status = 'pending'` gives BACKEND an efficient queue scan.
- A city can exist without coordinates and be fully usable for trip logging (GE-13). Map pins are suppressed in FRONTEND for pending cities.

## 2. Admin and Config Tables

All admin list tables share a consistent soft-delete pattern: `is_active = 0` hides the record from entry forms while preserving it on existing associations (AD-06).

### 2.1 trip_categories

```sql
CREATE TABLE trip_categories (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL UNIQUE,
    is_active   INTEGER NOT NULL DEFAULT 1  CHECK (is_active IN (0, 1)),
    created_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);
```

### 2.2 activities

```sql
CREATE TABLE activities (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL UNIQUE,
    is_active   INTEGER NOT NULL DEFAULT 1  CHECK (is_active IN (0, 1)),
    created_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);
```

### 2.3 companions

```sql
CREATE TABLE companions (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL UNIQUE,
    is_active   INTEGER NOT NULL DEFAULT 1  CHECK (is_active IN (0, 1)),
    created_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);
```

### 2.4 map_shading_config

State keys are fixed by the application. The user configures `display_name` and `color_hex` only. Defaults seeded from `_project/seed-data.txt`.

```sql
CREATE TABLE map_shading_config (
    state_key     TEXT PRIMARY KEY,  -- Fixed: 'active' | 'planned' | 'visited_once' |
                                     -- 'visited_once_planning' | 'visited_multiple' |
                                     -- 'visited_multiple_planning'
    display_name  TEXT NOT NULL,
    color_hex     TEXT NOT NULL,     -- e.g. '#2196F3' — validated by FRONTEND
    updated_at    TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);
```

**Notes:**
- 'never_visited' is not a row in this table — it has no shading by definition (MP-05).
- `state_key` values are constants in application code, not user-editable. Only `display_name` and `color_hex` are mutable via the admin panel (MP-04).

## 3. Trips

### 3.1 trips

```sql
CREATE TABLE trips (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    name             TEXT NOT NULL,
    start_date       TEXT NOT NULL,  -- ISO 8601 date: 'YYYY-MM-DD'
    end_date         TEXT NOT NULL,  -- ISO 8601 date: 'YYYY-MM-DD'
    status           TEXT NOT NULL DEFAULT 'planning'
                         CHECK (status IN ('planning', 'active', 'review_pending', 'locked')),
    photo_album_ref  TEXT,           -- URL or folder path reference; no file stored (PH-01)
    created_at       TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at       TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);

CREATE INDEX idx_trips_status     ON trips(status);
CREATE INDEX idx_trips_start_date ON trips(start_date);
CREATE INDEX idx_trips_end_date   ON trips(end_date);
```

**Notes:**
- `status = 'locked'` is enforced read-only in BACKEND. There is no separate `is_locked` boolean — the status field is the single source of truth (TR-06, TR-07).
- `start_date = end_date` is permitted (day trip, GE-11 coverage).
- No constraint prevents `end_date < start_date` in SQLite CHECK; BACKEND must validate this.

### 3.2 trip_categories_map

```sql
CREATE TABLE trip_categories_map (
    trip_id      INTEGER NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
    category_id  INTEGER NOT NULL REFERENCES trip_categories(id),
    PRIMARY KEY (trip_id, category_id)
);

CREATE INDEX idx_tcm_category ON trip_categories_map(category_id);
```

### 3.3 trip_companions_map

```sql
CREATE TABLE trip_companions_map (
    trip_id      INTEGER NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
    companion_id INTEGER NOT NULL REFERENCES companions(id),
    PRIMARY KEY (trip_id, companion_id)
);

CREATE INDEX idx_tcpm_companion ON trip_companions_map(companion_id);
```

### 3.4 trip_activities_map  *(trip-level activities)*

```sql
CREATE TABLE trip_activities_map (
    trip_id     INTEGER NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
    activity_id INTEGER NOT NULL REFERENCES activities(id),
    PRIMARY KEY (trip_id, activity_id)

```

## 4. Places (Trip-Place Associations)

### 4.1 trip_places

A trip_place record is created for each city visited within a trip. This is the "Place" entity from the BRD. Items and place-level activities reference this table.

```sql
CREATE TABLE trip_places (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    trip_id     INTEGER NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
    city_id     INTEGER NOT NULL REFERENCES cities(id),
    created_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at  TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    UNIQUE (trip_id, city_id)  -- a trip visits each city at most once
);

CREATE INDEX idx_trip_places_trip ON trip_places(trip_id);
CREATE INDEX idx_trip_places_city ON trip_places(city_id);  -- critical for cross-trip queries
```

**Notes:**
- The UNIQUE constraint on `(trip_id, city_id)` is intentional. Multiple stays in the same city on one trip are captured as multiple hotel items under the same trip_place, not as separate trip_place records.
- Cross-trip queries (IT-07 carry-forward, IT-09 city-level rating view) join on `city_id`.

### 4.2 trip_place_activities_map  *(place-level activities)*

Separate from trip-level activities. A user can tag activities to a specific city within a trip (TR-04).

```sql
CREATE TABLE trip_place_activities_map (
    trip_place_id INTEGER NOT NULL REFERENCES trip_places(id) ON DELETE CASCADE,
    activity_id   INTEGER NOT NULL REFERENCES activities(id),
    PRIMARY KEY (trip_place_id, activity_id)
);
```

## 5. Items

### 5.1 items  *(base table)*

```sql
CREATE TABLE items (
    id                    INTEGER PRIMARY KEY AUTOINCREMENT,
    trip_id               INTEGER NOT NULL REFERENCES trips(id) ON DELETE CASCADE,
    trip_place_id         INTEGER REFERENCES trip_places(id),  -- NULL = trip-level item
    item_type             TEXT NOT NULL
                              CHECK (item_type IN
                                  ('restaurant', 'hotel', 'flight',
                                   'car_rental', 'experience', 'note')),
    status                TEXT NOT NULL DEFAULT 'consider'
                              CHECK (status IN
                                  ('consider', 'confirmed', 'completed',
                                   'cancelled', 'next_time')),
    notes                 TEXT,            -- applies to all item types (IT-04)
    is_carried_forward    INTEGER NOT NULL DEFAULT 0  CHECK (is_carried_forward IN (0, 1)),
    carried_from_item_id  INTEGER REFERENCES items(id),  -- self-ref; NULL for original items
    created_at            TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now')),
    updated_at            TEXT NOT NULL DEFAULT (strftime('%Y-%m-%dT%H:%M:%SZ', 'now'))
);

CREATE INDEX idx_items_trip        ON items(trip_id);
CREATE INDEX idx_items_trip_place  ON items(trip_place_id);
CREATE INDEX idx_items_type        ON items(item_type);
CREATE INDEX idx_items_status      ON items(status);
CREATE INDEX idx_items_carried     ON items(carried_from_item_id)
    WHERE carried_from_item_id IS NOT NULL;
```

**Notes:**
- `trip_place_id` is NULL for items logged at trip level (not against a specific city).
- `is_carried_forward = 1` AND `carried_from_item_id IS NOT NULL` together identify a carry-forward item. The boolean allows simple filtering; the FK preserves lineage for display ("carried from [trip name]").
- `is_carried_forward` without `carried_from_item_id` is an invalid state — BACKEND must enforce.
- Experience and Note items use only this base table. No extension table is created for these types because they carry no type-specific structured fields.

### 5.2 item_flights

One row per item_id where `item_type = 'flight'`. Each leg is a separate item (FL-01).

```sql
CREATE TABLE item_flights (
    item_id             INTEGER PRIMARY KEY REFERENCES items(id) ON DELETE CASCADE,
    airline             TEXT,
    flight_number       TEXT,
    departure_airport   TEXT,   -- IATA code preferred but not enforced
    arrival_airport     TEXT,   -- IATA code preferred but not enforced
    departure_datetime  TEXT,   -- ISO 8601 datetime
    arrival_datetime    TEXT,   -- ISO 8601 datetime
    booking_reference   TEXT,
    seat                TEXT
);
```

### 5.3 item_hotels

```sql
CREATE TABLE item_hotels (
    item_id              INTEGER PRIMARY KEY REFERENCES items(id) ON DELETE CASCADE,
    property_name        TEXT,
    address              TEXT,

    check_out_date       TEXT,  -- ISO 8601 date; duration computed from these two (HT-02)

```

```sql
    provider          TEXT,
    pickup_location   TEXT,
```

### 5.5 item_restaurants

```sql
CREATE TABLE item_restaurants (

    name               TEXT,
    neighbourhood_area TEXT,
    cuisine_type       TEXT,
```

### 5.6 item_experiences

Added in schema v1.1. PO confirmed Experiences are rateable (EX-01, BRD v2.3).

```sql
CREATE TABLE item_experiences (
    item_id           INTEGER PRIMARY KEY REFERENCES items(id) ON DELETE CASCADE,
    rating            INTEGER CHECK (rating BETWEEN 1 AND 5),  -- NULL = unrated (EX-01)
    post_visit_notes  TEXT                                      -- NULL until reviewed (EX-01)
);

CREATE INDEX idx_item_experiences_rating ON item_experiences(rating);
```

**Note:** Experience items with no rating or post-visit notes do not require a row in this table. BACKEND should create the extension row lazily when the user first adds a rating or note to a completed Experience.

## 6. Key Queries for BACKEND Reference

### 6.1 All items at a city across all trips (city-level view, IT-09)

```sql
SELECT
    i.*,
    ir.name            AS restaurant_name,
    ir.rating          AS restaurant_rating,
    ih.property_name   AS hotel_name,
    ih.rating          AS hotel_rating,
    ie.rating          AS experience_rating,
    t.name             AS trip_name,
    t.start_date
FROM items i
JOIN trip_places tp ON tp.id = i.trip_place_id
JOIN trips t        ON t.id  = tp.trip_id
LEFT JOIN item_restaurants ir  ON ir.item_id = i.id
LEFT JOIN item_hotels      ih  ON ih.item_id = i.id
LEFT JOIN item_experiences ie  ON ie.item_id = i.id
WHERE tp.city_id = :city_id
  AND i.item_type IN ('restaurant', 'hotel', 'experience')
  AND i.status = 'completed'
ORDER BY COALESCE(ir.rating, ih.rating, ie.rating) DESC NULLS LAST;
-- Replace ORDER BY with WHERE COALESCE(...) >= :min_rating for filtering
```

### 6.2 Next time items eligible for carry-forward (IT-07)

```sql
SELECT
    i.*,
    ir.name AS restaurant_name,
    ih.property_name AS hotel_name,
    t.name  AS source_trip_name,
    t.end_date AS source_trip_end_date
FROM items i
JOIN trip_places tp ON tp.id = i.trip_place_id
JOIN trips t        ON t.id  = tp.trip_id
LEFT JOIN item_restaurants ir ON ir.item_id = i.id
LEFT JOIN item_hotels      ih ON ih.item_id = i.id
WHERE tp.city_id = :city_id
  AND i.status = 'next_time'
  AND t.status IN ('review_pending', 'locked')
ORDER BY t.end_date DESC;
```

### 6.3 Geocoding queue (GE-12)

```sql
SELECT id, country_code, region_id, name, geocode_attempted_at
FROM cities
WHERE geocode_status = 'pending'
ORDER BY geocode_attempted_at ASC NULLS FIRST;  -- oldest attempts first
```

## 8. Entity Relationship Summary

```
countries ──< regions ──< cities
                              │
                              └──< trip_places >──── trips
                                        │                │
                                        │                ├──< trip_categories_map >── trip_categories
                                        │                ├──< trip_companions_map  >── companions
                                        │                └──< trip_activities_map  >── activities
                                        │
                                        ├──< trip_place_activities_map >── activities
                                        │
                                        └──< items ──── item_flights
                                                   ├─── item_hotels
                                                   ├─── item_car_rentals
                                                   ├─── item_restaurants
                                                   └─── item_experiences

map_shading_config  (standalone config table)
trip_categories     (admin list)
activities          (admin list)
companions          (admin list)
```

Legend: `──<` = one-to-many, `>──` = many-to-one, `>──<` = junction (many-to-many)
