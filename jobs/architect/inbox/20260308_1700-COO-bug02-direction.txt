BACKGROUND

QA static review found that RegionLayer.tsx applies feature-state using
integer DB region_id values (e.g. id: 1) but the GeoJSON regions.json uses
ISO 3166-2 string codes (e.g. "US-CA") as feature IDs via promoteId="iso_3166_2".
These never match. No region shading will ever render.

Root cause: the regions table has no iso_3166_2 column. This is a schema
defect — the blueprint specified ISO 3166-2 matching but it was not implemented.
