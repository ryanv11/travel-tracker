SECTION 2 — GOVERNANCE CHANGES — IN EFFECT IMMEDIATELY

The following operational changes have been formally adopted as project delivery
governance. They apply from this date forward.

OP-01 — QA engagement starts earlier
  QA involvement begins before frontend completion. Test design should start
  from the API contract and workflow rules while development is still in progress.
  The current situation (4 MAJORs found in static review, live testing blocked
  by environment) is exactly what this prevents. We are correcting the model now.

OP-02 — Contract-based test design
  Create test scenarios from the API contract and key state transitions,
  including negative-path and lock-state tests. Do not wait for a completed
  frontend to begin this work.

OP-03 — Shared regression pack
  Informal developer unit checks are not sufficient. Build a shared regression
  pack covering: carry-forward, trip lock/unlock, city creation, retry behavior,
  and structured list changes. This pack should be runnable independently of
  any single team's tooling.

OP-04 — Definition of done uplift
  Business-critical flows require unit tests + scenario validation before a
  story is considered complete. BUG-03 (carry-forward race condition) is a
  clear example of what scenario testing would have caught at development time.

OP-05 — Backend hotspot sign-off
  Backend corrections affecting carry-forward, city patching, or write protection
  are release-relevant and require explicit COO sign-off before frontend closeout.
  This is now standing procedure.
