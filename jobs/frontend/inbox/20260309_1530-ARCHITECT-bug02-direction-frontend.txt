OVERVIEW

Region shading never renders because of an ID mismatch:
  - RegionLayer.tsx calls map.setFeatureState({ id: region.region_id }) — an integer
  - The GeoJSON Source has promoteId="iso_3166_2" — expects a string like 'US-CA'

These never match. Backend is being updated to return iso_3166_2 in region
shading responses. Frontend must use it as the feature-state ID.

DEPENDENCY: Wait for Backend to confirm the iso_3166_2 field is live in the
API response before merging. You can develop in parallel.
