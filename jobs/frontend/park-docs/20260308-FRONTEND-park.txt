=== KEY DECISIONS ===

1. Separate tsconfig.frontend.json
   The root tsconfig.json uses "moduleResolution": "NodeNext" for the backend.
   Vite requires "bundler". Created separate tsconfig rather than breaking backend.

2. Frontend types in api.ts (not imported from drizzle schema)
   Avoided importing drizzle-orm/sqlite-core into the browser bundle by defining
   standalone TypeScript interfaces in src/frontend/types/api.ts.

3. C1 carry-forward fully implemented
   The BACKEND inbox message confirmed the carry-forward POST endpoint was live.
   CarryForwardModal is fully wired, not left as a TODO.

4. Item status follows Zod schema (not API reference doc)
   The API reference doc uses "booked"/"skipped" — inconsistent with the Zod
   zItemStatus enum (consider/confirmed/completed/cancelled/next_time).
   Frontend uses the schema as authoritative. Flagged to COO as FLAG-F1.

5. onCountryClick is optional on MapView
   MapPage has no current use for country selection; made it optional with ?. syntax.
